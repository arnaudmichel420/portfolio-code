Cinema

  • Field :
  • Web development
  • Brief :
  • The goal of the project is to develop a web platform that allows users to reserve cinema tickets for screenings online. The project will include :
    • A front-office accessible to the public for viewing films and booking tickets while log-in.
    • A back-office reserved for administrators to manage films, screenings and track reservations.
    • An API with a CRUD enabling the two clients to interact with the database.

Architecture

We have an MVC (Model View Controller) architecture :

  • On the database side, I opted for MySQL because it's the one I'm the most comfortable with.
  • On the server side, I'm using the Sequalize ORM to simplify SQL queries by simply manipulating JavaScript objects. The server is in Node.js because I didn't know Symfony at the time.
  • On the client side, I'm using Vue 3, which is one of the easiest thick clients to learn. The client and server talk to each other using an API.

Process :

Since the database schematics were provided, I started by creating the models with it and the migrations which you have to do manually on Sequalize. Then I used my wireframe to know which routes I needed to create on every page. I make sure to use middleware on the routes that need users to be log-in or have admin roles. I tried my routes with Postman and replicated them in a class on the front to be easily accessible everywhere on the code.

On the client side, I used Vue component to display movies and screenings, recovered with an automatic API request with the current day. Once the customer chooses his screening, he is taken to the booking page, where he can select a seat and visualize the ones already taken. When the booking is completed the customer is taken to his account where he can see all his previous reservations and the ones coming which are cancellable.

Finally, I created a back office for the staff to add new movies and screenings, to make things easier I coded a custom select component with a filtering input to reduce errors and improve efficiency.
